Cryptographic API Usage Instructions

This page provides instructions on how to use the cryptographic APIs for key generation, encryption, decryption, hash generation, and hash verification. The APIs are built using FastAPI and the Cryptography library in Python. You can test these APIs using tools like Postman or any HTTP client.

Base URL: https://kurrshanthv.eu.pythonanywhere.com

1. Key Generation Endpoint

POST /generate-key

Description: Generates a cryptographic key based on the specified type and size.

Request Body (JSON):

{
    "key_type": "AES",  // or "RSA"
    "key_size": 256     // for AES: 128, 192, 256; for RSA: 2048, 3072, etc.
}

Response (JSON):

{
    "key_id": "unique-key-id",
    "key_value": "base64-encoded-key"
}

Note: Save the key_id for encryption and decryption operations.

2. Encryption Endpoint

POST /encrypt

Description: Encrypts the provided plaintext using the specified key and algorithm.

Request Body (JSON):

{
    "key_id": "unique-key-id",
    "plaintext": "message-to-encrypt",
    "algorithm": "AES"  // or "RSA"
}

Response (JSON):

{
    "ciphertext": "base64-encoded-ciphertext"
}

3. Decryption Endpoint

POST /decrypt

Description: Decrypts the provided ciphertext using the specified key and algorithm.

Request Body (JSON):

{
    "key_id": "unique-key-id",
    "ciphertext": "base64-encoded-ciphertext",
    "algorithm": "AES"  // or "RSA"
}

Response (JSON):

{
    "plaintext": "original-message"
}

4. Hash Generation Endpoint

POST /generate-hash

Description: Generates a hash for the given data using the specified hashing algorithm.

Request Body (JSON):

{
    "data": "message-to-hash",
    "algorithm": "SHA-256"  // or "SHA-512"
}

Response (JSON):

{
    "hash_value": "base64-encoded-hash",
    "algorithm": "SHA-256"
}

5. Hash Verification Endpoint

POST /verify-hash

Description: Verifies if the provided hash matches the data using the specified algorithm.

Request Body (JSON):

{
    "data": "message-to-verify",
    "hash_value": "base64-encoded-hash",
    "algorithm": "SHA-256"  // or "SHA-512"
}

Response (JSON):

{
    "is_valid": true,  // or false
    "message": "Hash matches the data."  // or "Hash does not match the data."
}
